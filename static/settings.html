Recorder Settings

Configure the defaults used when you start a recording, plus the advanced capture parameters that shape tab, camera, microphone, and chunking behavior.

Basic

Choose where finished recordings go by default. Drive uploads happen after capture stops.
Sets whether your microphone is ignored, mixed into the tab file, or saved as a separate file by default.
When enabled, the extension asks for your camera and writes it as its own recording file by default.
Select the preferred camera capture size. When Meet camera is off, separate camera recording can enforce this preset more strongly. Otherwise Chrome can still deliver a lower resolution.

Professional

Higher bitrate preserves more camera detail but increases file size and upload time.
Controls the requested camera motion smoothness. Higher values can use more bandwidth and CPU.
Sets the floor when adaptive camera bitrate is enabled so video quality does not drop below this level.
Sets the requested maximum tab capture size. Chrome may still deliver a different size depending on the page, display scale, and browser limits.
Caps the preferred tab frame rate. Chrome tab capture tops out at 30 FPS, and lower values can reduce CPU use and file size.
Asks Chrome to reduce playback echo leaking back into the microphone stream.
Requests background-noise cleanup on the microphone track before it is recorded.
Lets Chrome automatically boost or lower microphone volume while capturing audio.
Controls how often MediaRecorder flushes normal recording chunks to storage during simple runs.
Used when extra streams are active and the performance flag enables longer recorder chunk intervals.
